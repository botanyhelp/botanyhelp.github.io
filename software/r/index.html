R

Learning R programming is best achieved with Robert Kabacoff's R In Action, 2nd Edition. Garret Grolemund's Hands-On Programming with R is less celebrated but rewards experienced programmers by uncloaking lots of R's strangeness. Most programmers should be using Rstudio for development especially because it makes learning easier. Within R, you can do these things (you'll need an internet connection for that first line):

  • install.packages("swirl")
  • library(swirl)

Going through the swirl tutorial in R provides efficient learning for new and experienced R programmers, especially if you want learning by doing.

R is for statistics, you've probably heard, and biostatistics. The edX online learning platform has an excellent series of courses in R programming, statistics, biology and data analysis. Learn more at edX and at the book for these courses:

Rufosightings R package

Install it like this: 

library(devtools)
install_github("botanyhelp/Rufosightings")
library(Rufosightings)

Then you will have the Rufosightings data frame. It has 55633 rows and 9 variables and can be slow to query. I wrote this package using data that I got when reading Machine Learning For Hackers. This data is no longer easy to get using the mechanism described in the book. The package has only one function, ufo_byexactdate(). All of the excellent reports on in the R data frame that comes with the package. Anyone can install the Rufosightings from its location on github and try it out like this:

library(devtools)
install_github("botanyhelp/Rufosightings")
library(Rufosightings)
ufo_byexactdate()
ufo_byexactdate(20100704)
dim(Rufosightings)
Rufosightings[Rufosightings$sighted_at_year=="1950",]
Rufosightings[Rufosightings$sighted_at_year=="1942",]$description
Rufosightings[Rufosightings$sighted_at_year=="1942",]$duration
Rufosightings[Rufosightings$sighted_at_year=="1942",]$shape

Because I collected this data using web crawling python, there are lots of HTML artifacts in the data. A bigger problem is that there are some rows where the column shifted. You can see this with 

levels(Rufosightings$shape)

There is not supposed to be a shape called "5 minutes". That value for that row should have gone into the duration column. Hope to fix someday but its been a long time.

Also notice that the values in location_city and shape all start with a space character. (Sorry about that, hope to fix it). Therefore, when querying, every location_city value must be prepended with a space character:

Rufosightings[Rufosightings$location_city==" Slickville",]

Life with mediocre data packages like Rufosightings is better with dplyr. Here we get a better idea of the data and we also look for the description of cigar shaped UFOs seen in Miami:

library(dplyr)
glimpse(Rufosightings)
select(Rufosightings,c(location_city,shape,description)) %>% filter(location_city==" Miami") %>% filter(shape==" cigar")
